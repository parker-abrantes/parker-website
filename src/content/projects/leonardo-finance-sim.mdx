## Problem

Defense contract finance involves overlapping cost accounting standards, contract types (CPFF, FFP, T&M), earned value metrics, and DCAA audit requirements. There's no lightweight tool for students or early-career engineers to explore how these systems interact.

## What I Built

A full-stack web application that models:

- **Contract types** — Cost Plus Fixed Fee, Firm Fixed Price, Time & Materials
- **Earned value metrics** — CPI, SPI, EAC, VAC, TCPI
- **Burn rate visualization** — budget vs. actuals over time via Recharts
- **DCAA audit trail awareness** — documentation-first cost tracking
- **CSV export** — for external analysis

Backend: Python + FastAPI + SQLite. Frontend: React + Recharts.

## What I Learned

Earned value management is deceptively simple on paper and surprisingly subtle in practice. The gap between planned value and earned value can look identical in two projects with completely different root causes — one is ahead of schedule, one is burning budget on rework.
